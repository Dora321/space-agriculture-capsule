## 一句话

一个 **ESP32 全自治种植舱 + 树莓派/云端增强**的 IoT 系统，叙事是"在轨育种验证平台的地面雏形"。

**铁律**：ESP32 断了会死作物 → **必须自治**；树莓派 / 云 / AI 断了只是变笨/看不到 → **可降级**。

## 架构总图（双层）

```
┌─────────── ① ESP32 飞控（下位机）·· 只要有电就跑 ───────────┐
│ SENSE   土壤/光照/温湿/按键                                   │
│ THINK   本地规则引擎(兜底) + 采纳树莓派 advice(在线优先)       │
│ ACT     水泵/补光灯/WS2812灯条/OLED                           │
│ SAFE    安全护栏：温度 / 单次时长 / 限频                       │
└───────────────────────┬──────────────────────────────────────┘
            UART2 115200  │  ↑ report / pong      ↓ ping / advice
            JSON-over-Line│
┌───────────────────────▼─────── ② 树莓派（上位机）────────────┐
│ serial_gateway.py（systemd 自启）                            │
│  · 收 ESP32 report → 转发云端大屏                             │
│  · 生成 advice 发回 ESP32（规则 / 后续接 DeepSeek）            │
│  · 负责联网、大屏、AI、（后续）数据库/视觉                     │
└───────────────────────┬──────────────────────────────────────┘
              HTTP        │  (树莓派联网)
┌───────────────────────▼──────────────────────────────────────┐
│ ③ 云端 / PC 地面站                                            │
│  dashboard_server.py :8790  ⇄ 浏览器轮询 ⇄  实时大屏          │
│  ai_proxy.py :8787  →  DeepSeek（替 ESP32 扛 TLS 的中间人）    │
└───────────────────────────────────────────────────────────────┘
```

## 两层大脑 / 降级链（最容易搞混的一点）

系统有两个"决策者"，不是二选一，而是**从聪明到保命的两层兜底**——上层挂了下层接住（2026-05-31 移除了多余的树莓派阈值规则中间层，DeepSeek + ESP32 本地规则已足够）：

```
① DeepSeek（树莓派调用，最聪明）  看生长阶段/日照/作物参数，还给育种观察
        ↓ 超时 / 没网 / 没 key → 网关不发 advice
② ESP32 本地规则（板上，永远在）  utils.local_fallback_decision：干旱→低温→浇水→补光→高温→正常
        ↑ DeepSeek 失败、UART 断、树莓派没了，都回到这层
```

| 大脑 | 跑在哪 | 是什么 | 聪明度 |
|---|---|---|---|
| DeepSeek | 树莓派调用云端 | 大模型 | ⭐⭐⭐ |
| 本地规则 | ESP32 | 固定优先级 if | ⭐ |

> 真正"聪明"的是 DeepSeek；ESP32 本地规则是断网断 Pi 都能种活的保命兜底。
> 这套分层咬合"深空自治"叙事：**联网时精细，断联时可靠**。

## 一条数据的旅程

```
传感器采样(60s)
  → ESP32 本地规则先算一个兜底决策
  → 若有树莓派在线 advice：经安全护栏后优先采用
  → 执行动作(水泵/补光) + WS2812 信号广播 + OLED 刷新
  → 经 UART 把 report 发给树莓派 → 树莓派转发云端大屏
```

## 运行模式与演进状态

系统只有**一种**架构：双层（ESP32 ──UART──▶ 树莓派 ──WiFi──▶ 云端）。早期的单层 WiFi 老路（ESP32 自己联网+调 DeepSeek）已移除。

**为什么转双层**：ESP32 同时开 WiFi+OLED+UART 堆内存不够（OOM），把联网/大屏/AI 整个甩给树莓派。

**演进状态（2026-05-30，分支 `refactor/remove-wifi-only-path`，133 测试绿）**：
- ✅ ESP32 去 WiFi/AI/telemetry，删 `wifi_client`/`telemetry`/`ai_client`，固件固定走双层
- ✅ DeepSeek 从 ESP32 搬到树莓派（`tools/pi_advisor.py` + `serial_gateway --ai-advice`）
- ✅ 完整 ①DeepSeek（Pi）→②ESP32本地 两层降级成型（2026-05-31 去掉多余的 Pi 阈值规则中间层）
- ⏳ 待办：`SPACEFARM_AI_API_KEY` 配到 Pi systemd；实机验证 AI 往返（卡在地线）；合并 master 前确认硬件链路稳（砍老路后无 WiFi 兜底）
