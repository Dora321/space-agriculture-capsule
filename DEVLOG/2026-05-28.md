# 2026-05-28 核心架构升级：从「执行器驱动」到「决策驱动」

## 今日目标

实现 **Decision Plane / Action Plane 分离架构**，将系统从单纯的"传感器→动作"升级为"决策层广播多维诊断信号，执行层仅响应有硬件支撑的动作"。

核心叙事：「决策能力和执行能力是分离的」——即使执行器未就位（如缺肥信号没有营养泵），诊断信息也已通过灯条广播。

## 遇到的问题

1. **决策优先级错误**：光照判断在土壤干燥之前，导致 test_demo_decision_and_recovery_cycle 返回 'light' 而非 'water'。修复：重排优先级为 关键干旱→低温→土壤干旱→光照→高温→正常。

2. **AI 门控测试失败**：`start_time=0` 导致 uptime_sec 巨大，触发 `uptime_sec > 43200 and sun_hours < light_hours[0]` 检查返回 'light'。修复：设置 `start_time = time.time() - 3600`。

3. **Edit 工具缩进不匹配**：utils.py 混合 tab/space 缩进导致 old_string 无法匹配。修复：用 Write 工具完整重写文件。
