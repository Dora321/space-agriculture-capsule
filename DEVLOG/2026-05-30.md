# 2026-05-30 调试日：树莓派双层架构阶段二

## 背景

根据《架构设计-树莓派上位机.md》，系统升级为“ESP32 舱内飞控 + 树莓派载荷计算机/地面站后端”的双层架构。原则保持不变：树莓派增强但不接管，ESP32 保留传感器、执行器、本地规则和安全护栏。

## 完成事项

### #35 ESP32 UART 主循环接入

新增 `esp32_firmware/uart_link.py` 之后，本次把它真正接进运行链路：

- `config.py.example` 新增 `UART_ENABLED / UART_ID / UART_BAUD / UART_TX_PIN / UART_RX_PIN / UART_OFFLINE_TIMEOUT_MS`
- `main.py` 新增 `_init_uart_link()`，启用后使用 UART2：GPIO17 TX、GPIO16 RX、115200 8N1
- `loop_runtime.py` 新增 `uart_poll` 和 `uart_send_report` 注入点
- 每轮传感器采样后发送 `report`
- 100ms 子循环中持续轮询串口，及时响应树莓派 `ping` 并接收 `advice`
- `state.py` 新增 `pi_online` 和 `pending_pi_decision`

默认 `UART_ENABLED=False`，现有 WiFi-only 比赛路径不受影响。实机双层模式使用：

```python
UART_ENABLED = True
UART_SKIP_WIFI = True
UART_RXBUF = 256
STARTUP_MENU_ON_BOOT = True
```

### #36 Pi advice 下发与 ESP32 护栏

ESP32 现在会把树莓派 `advice` 转换为内部 decision，并在决策周期优先采用在线 Pi 建议：

- Pi 在线且存在新 advice：`last_decision_source = "pi"`
- Pi 超过 30 秒无消息：丢弃 stale advice，回到本地规则
- Pi 建议浇水时仍受 ESP32 温度护栏约束
- 动作时长仍按 ESP32 本地上限裁剪

同时修复一个顺手发现的问题：`action_runtime.py` 原先对补光灯也使用 `PUMP_MAX_RUN_SEC` 裁剪，本次改为 light 动作用 `LIGHT_MAX_RUN_SEC`。

### #37 树莓派 serial_gateway 验收模式

`tools/serial_gateway.py` 保留纯协议层 `GatewayCore`，并增强硬件运行层：

- `--auto-advice`：收到 ESP32 `report` 后用保守规则生成建议
- `--test-advice water --test-duration 8`：收到第一条 report 后只下发一次固定 advice，方便验收“树莓派能让 ESP32 浇水”
- 持续发送 `ping`，ESP32 自动回 `pong`
- 继续把 ESP32 `report` 转成 dashboard `/api/state`

### #38 UART 实机初始化与内存顺序修正

接好 ESP32 GPIO17/16 与树莓派 GPIO15/14/GND 后，开始实机验证 UART。排查过程：

1. 设备端原本没有 `UART_ENABLED`，仅上传了 `uart_link.py`，默认不会启动 UART。
2. 打开 `UART_ENABLED=True` 后，UART 放在 OLED 初始化之后会失败：

```text
E uart: UART driver malloc error
[UART] init skipped: [Errno 1] EPERM: ESP_FAIL
```

3. 把 UART 提前到 WiFi 之后、OLED 之前，UART 可以初始化，但随后 OLED/I2C 初始化触发 MicroPython 底层崩溃并重启。
4. 结论：ESP32 同时启用 WiFi 驱动、UART2、OLED/I2C 时堆内存压力过高，双层架构应把联网职责移到树莓派。
5. 增加 `UART_SKIP_WIFI=True`，UART 模式下 ESP32 跳过 WiFi，再先初始化 UART2，随后初始化 OLED。实机日志确认稳定：

```text
[WiFi] Skipped: UART mode uses Raspberry Pi networking
[UART] Pi link initialized
[Display] OLED initialized successfully
[Init] Starting plant selection...
```

当前 ESP32 侧状态：

- UART2 初始化成功
- OLED 正常
- 开机植物选择和天数选择恢复
- ESP32 WiFi 被跳过是预期行为
- 树莓派端 `/dev/serial0` report/ping/pong/advice 全链路仍需继续验收
