# 三人联排脚本 · 3分钟完整答辩流程

> **使用方法**：打印1份，教练拿着计时。每次联排按此脚本走，精确到秒。
> **交接规则**：一个人说完最后一句话，下一个人必须在2秒内接上。

---

## 🕐 时间轴总览

```
0:00 ─── 队员3开场 ──────────────────────── 0:30
0:30 ─── 队员1硬件演示 ─────────────────── 1:30
1:30 ─── 队员2软件演示 ─────────────────── 2:30
2:30 ─── 队员3总结 ─────────────────────── 3:00
```

---

## 📝 逐秒脚本

### 【0:00-0:30】队员3 · 立意（30秒）

> **舞台位置**：站在评委正前方，面向评委
> **配合**：右手自然指向KT板

**话术**（逐字稿）：

```
"各位评委好！我们的项目是《太空农业智能种植舱》。

（停顿1秒，看评委）

为什么做这个项目？让我们看一段历史：

公元前8000年，人类驯化野生小麦，周期10000年；
1865年，孟德尔发现遗传规律，周期缩短到10年；
1973年，袁隆平杂交水稻，周期5到10年；
1987年，中国开始太空育种，但种子上天绕一圈，
回到地面还要种5到8代才能筛出好品种。

问题来了：太空诱变频率是地面的3到4倍，
但反馈周期被'搭载机会'卡死了。

我们的解决方案是：
让种植和筛选本身就发生在太空舱里！

（指向队员1）

接下来请我的队友介绍硬件实现。"
```

**交接手势**：说完最后一句话，右手伸向队员1方向，队员1走上前。

---

### 【0:30-1:30】队员1 · 硬件演示（60秒）

> **舞台位置**：走到展台前，拿起电路板
> **配合**：队员3退到KT板旁边，随时指向KT板

**话术**（逐字稿）：

```
【0:30-0:50】自我介绍 + 零件介绍（20秒）

"大家好，我是硬件工程师。

我们的种植舱用ESP32做主控，成本只要140元。

（拿起ESP32）
这是ESP32主控板，是整个系统的大脑；
（拿起土壤传感器）
这是土壤湿度传感器，插在土里测水分；
（拿起DHT11）
这是DHT11温湿度传感器，测温度和湿度；
（指继电器）
这是继电器模块，是电子开关，控制水泵。"

【0:50-1:15】断网自治演示（25秒）

"现在我演示断网自治能力。

（拔掉WiFi/USB线，停顿2秒）

大家看OLED屏幕，已经显示'本地模式'，
系统自动切换到本地规则引擎，继续正常浇水。

这就是我们的断网自治能力，
模拟火星延迟场景——火星到地球有4到24分钟延迟，
不可能实时遥控。

（插回WiFi/USB线，等3秒）

WiFi恢复后，系统自动切回AI模式。"

【1:15-1:30】容错机制（15秒）

"我们系统有三层保护：
第一，看门狗，程序卡死会自动重启；
第二，内存监控，不够就跳过AI；
第三，传感器失效会自动用安全值。

（指向队员2）

接下来请我的队友介绍软件和AI决策。"
```

**交接手势**：说完最后一句话，右手伸向队员2方向，队员2走上前。

---

### 【1:30-2:30】队员2 · 软件演示（60秒）

> **舞台位置**：走到笔记本电脑旁边
> **配合**：队员1退到展台旁边，队员3继续站在KT板旁边

**话术**（逐字稿）：

```
【1:30-1:55】AI决策流程（25秒）

"我是软件工程师，负责AI决策系统。

我们的核心创新是'双决策引擎'：

本地规则引擎，像条件反射，快但简单；
AI决策引擎，像大脑思考，慢但聪明。

系统不是随时都请AI，而是三个条件触发：
第一，阈值事件，比如土壤太干了；
第二，环境变化，比如突然阴天；
第三，定时复核，每2小时AI检查一次。

这样节省电力和带宽，适合太空场景。"

【1:55-2:15】Web大屏介绍（20秒）

"这是我们的实时监控大屏：

（手指向屏幕左边）
左边是指标仪表盘，显示温度、湿度、光照、水位；
（手指向屏幕中间）
中间是决策流水线，展示传感器数据怎么变成执行动作；
（手指向屏幕右边）
右边是执行记录，记录什么时间、做了什么、为什么。

所有数据每5秒更新一次。"

【2:15-2:30】测试保障（15秒）

"我们写了87个自动化测试用例，
覆盖传感器失效、断网、内存不足等20多种故障。

所有测试全部通过，保证系统稳定。

（退后一步，让出C位给队员3）

请我们的项目发言人做总结。"
```

**交接手势**：说完最后一句话，退后一步，队员3走上前。

---

### 【2:30-3:00】队员3 · 总结（30秒）

> **舞台位置**：回到评委正前方
> **配合**：队员1和队员2分别站在两侧，配合指向KT板

**话术**（逐字稿）：

```
【2:30-2:50】三个创新点（20秒）

"总结一下，我们的项目有三个创新点：

第一，双决策引擎。
AI和本地规则互相备份，断网也能正常工作，
适合深空探索。

第二，低成本。
整套硬件140元，比人工气候室便宜100倍，
可以批量部署，做平行实验。

第三，教育价值。
我们把复杂的太空育种问题，
简化成一个中小学生能懂、能做的项目。"

【2:50-3:00】收尾（10秒）

"一万年前，我们的祖先第一次留下了最好的麦穗。
1987年，中国第一次把水稻种子送上太空。
今天，我们把这套系统的地面雏形做了出来。

希望十年后，有人把它送上空间站，
让人类的育种反馈周期，再缩短一次。

谢谢各位评委！"
```

**收尾动作**：说完"谢谢各位评委"，鞠躬15度。

## 🆘 应急预案

### 场景1：总时间超过3分钟

**判断标准**：2:30时队员3还没开始总结

**应急措施**：
- 队员3压缩总结，只说：
  ```
  "总结：三个创新点——双引擎、低成本、教育价值。
  今天它是课堂作品，明天它可能是火星基地的菜园子。
  谢谢各位评委！"
  ```
- 目标：30秒内说完

---

### 场景2：硬件故障（ESP32死机、OLED不亮）

**应急措施**：
- 队员1说："我们的系统有看门狗机制，3秒后会自动重启。请大家稍等。"
- 等待5秒，如果还没好：
  - 队员1说："我们现在切换到备用演示方案。"
  - 队员2打开Web大屏的"演示模式"
  - 继续软件演示部分

---

### 场景3：Web大屏打不开

**应急措施**：
- 队员2说："我们准备了截图。"
- 打开桌面上的"大屏截图"文件夹
- 用截图代替实时演示
- 队员3说："我们的Web大屏是实时数据看板，
  今天因为网络原因用截图展示。"

---

### 场景4：评委提前打断提问

**应急措施**：
- 谁被问谁回答
- 如果是硬件问题 → 队员1回答
- 如果是软件问题 → 队员2回答
- 如果是立意问题 → 队员3回答
- 回答完，队员3说："我继续刚才的介绍。"然后回到脚本

---

### 场景5：队员严重忘词（超过10秒说不出话）

**应急措施**：
- 下一个队员立刻补位，说："我补充一下..."
- 帮忘词的队员起头，然后让TA继续
- 如果完全接不上，直接跳到下一个环节
