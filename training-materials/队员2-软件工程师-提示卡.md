## 第一步：Web大屏3个区域（指认）

### 区域1：左边 - 指标仪表盘

**记住这句话**：
```
"左边是指标仪表盘，显示温度、湿度、光照、水位。"
```

**要指的地方**（评委看大屏时，你手指向这些位置）：
- 左上角：温度仪表
- 左中：湿度仪表
- 左下角：光照/水位

**训练方法**：
- 打开`groundstation.html`
- 用手指依次指向这3个位置
- 边指边说："这是温度...这是湿度...这是光照..."
- 练20遍，形成肌肉记忆

---

### 区域2：中间 - 决策流水线

**记住这句话**：
```
"中间是决策流水线，展示传感器数据怎么变成执行动作。"
```

**要指的地方**：
- 中间最上面：传感器数据输入
- 中间中间：本地规则 / AI决策（两个框）
- 中间下面：执行动作输出

**训练方法**：
- 打开大屏，手指从左到右移动
- 说："传感器数据 → 本地规则 → AI优化 → 执行动作"
- 练20遍

---

### 区域3：右边 - 执行记录

**记住这句话**：
```
"右边是执行记录，记录什么时间、做了什么、为什么。"
```

**要指的地方**：
- 右边整个区域：表格/日志
- 表头：时间 / 动作 / 原因

**训练方法**：
- 打开大屏，手指向右边的表格
- 说："这是我们的执行日志，每次浇水都会记录"
- 练20遍

## 第二步：4个操作（练到不用思考）

### 操作1：打开大屏

**步骤**：
1. 双击`groundstation.html`
2. 等待5秒，页面加载完成
3. 说："这是我们的实时监控大屏"

**⚠️ 关键**：
- 提前打开浏览器，不要现场双击
- 如果打不开，说："我们准备了截图"（备用方案）

---

### 操作2：刷新页面

**步骤**：
1. 按F5，或者点刷新按钮
2. 等待3秒，页面重新加载
3. 说："数据每5秒自动更新"

**⚠️ 关键**：
- 提前准备好"演示模式"的话术："这是演示模式，真实数据需要连接硬件"

---

### 操作3：指认3个区域（配合话术）

**步骤**：
1. 说："左边是指标仪表盘..." → 手指向左边
2. 说："中间是决策流水线..." → 手指向中间
3. 说："右边是执行记录..." → 手指向右边

**⚠️ 关键**：
- 手指要稳，不要晃
- 停顿2秒，让评委看清

---

### 操作4：演示OLED三页翻页

**步骤**：
1. 拿起电路板，指向OLED屏幕
2. 说："我们的OLED有三页信息，用红黄键翻页——"
3. 按黄键 → 第1页：土壤湿度大字+作物图标
   → 说："第一页，3米外也能看清土壤湿度"
4. 按黄键 → 第2页：生长进度条+施肥建议
   → 说："第二页，告诉你现在该施什么肥"
5. 按黄键 → 第3页：剩余内存+Pi在线状态
   → 说："第三页，Pi是否在线一目了然"
6. 按蓝键 → 进入菜单系统
   → 说："蓝键单击进入菜单，可以切换植物、设置天数"

**⚠️ 关键**：
- 如果电路板没电，说："三页布局我用截图展示"
- 提前准备三页截图

**训练方法**：
- 实际操作电路板10遍，记住翻页顺序
- 边按边说，形成肌肉记忆

## 第三步：背3段话术

### 话术1：两层决策设计（约25秒）

```
"我是这座育种站的站长，张煦涵——舱的大脑由我管。

我们的核心是'两层决策'：
第一层，AI（DeepSeek跑在树莓派上）；
第二层，ESP32本地规则接管。

两层决策的意义：
AI给建议，ESP32做判断——
就算AI不在线，ESP32自己也能让植物活着。
这就是'自治优先'的设计。"
```

**背诵技巧**：
- 先报身份："我是站长，舱的大脑我管"
- 记住关键词：两层决策 → AI → 本地规则
- 最后一句："挂了上层，下层接住"

---

### 话术2：地面站大屏介绍（约15秒）

```
"这是我每天盯的地面站大屏——
左边是当前样本：种的什么、第几天、长到哪个阶段；
中间是AI决策链：它看到什么、怎么判断、给什么建议；
右边是执行器和事件流：水泵、补光灯什么时候动过，一笔笔都记着。

数据从舱里实时传过来，浏览器自动刷新。"
```

**背诵技巧**：
- 边说边指（左边 → 中间 → 右边）
- 记住"左中右"：样本 → AI决策链 → 执行器/事件流（和大屏实际布局一致）

---

### 话术3：测试保障（约10秒）

```
"我们写了133个自动化测试用例，
覆盖30多种故障场景，全部通过。
这是我们的质量保证。"
```

**背诵技巧**：
- 记住数字：133个测试、30多种场景

## 第四步：可能的Q&A（4个问题）

### Q1: 你们的AI决策真的比定时浇水好吗？

**回答**：
```
"我们做过对比：
定时浇水存活率85%，阈值控制92%，AI优化95%。
AI的优势是动态调整——
比如阴天自动延长补光，晴天减少浇水。"
```

---

### Q4: 你们用的什么AI模型？

**回答**：
```
"我们用DeepSeek V4 Flash，跑在树莓派上。
ESP32不联网，通过UART发送数据给树莓派，
树莓派调AI后返回种植建议。
ESP32只管传感器和执行器，通信和AI全交给树莓派。"
```

---

### Q5: 你们的代码是自己写的吗？

**回答**：
```
"硬件固件是我们自己写的，用MicroPython。
AI模型调用现成API，我们负责判断逻辑。
总共19个模块，3000多行代码，全部开源在GitHub上。"
```

---

### Q6: 菜单系统怎么用？

**回答**：
```
"菜单有三项：植物选择、天数设置、系统信息。
4个键操作：红键上翻、黄键下翻、绿键确认、蓝键返回。
逻辑和手机设置一样，小学生也能用。"
```

**⚠️ 注意**：
- 如果评委要现场看，说："我打开电路板演示"
