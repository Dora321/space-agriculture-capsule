## 第一步：Web大屏3个区域（指认）

### 区域1：左边 - 指标仪表盘

**记住这句话**：
```
"左边是指标仪表盘，显示温度、湿度、光照、水位。"
```

**要指的地方**（评委看大屏时，你手指向这些位置）：
- 左上角：温度仪表
- 左中：湿度仪表
- 左下角：光照/水位

**训练方法**：
- 打开`contest-demo-dashboard.html`
- 用手指依次指向这3个位置
- 边指边说："这是温度...这是湿度...这是光照..."
- 练20遍，形成肌肉记忆

## 第三步：背3段话术（共约200字）

### 话术1：AI决策流程（40秒）

```
"我是软件工程师，负责AI决策系统。

我们的核心创新是'双决策引擎'：

本地规则引擎，像条件反射，快但简单；
AI决策引擎，像大脑思考，慢但聪明。

系统不是随时都请AI，而是三个条件触发：
第一，阈值事件，比如土壤太干了；
第二，环境变化，比如突然阴天；
第三，定时复核，每2小时AI检查一次。

这样节省电力和带宽，适合太空场景。"
```

**背诵技巧**：
- 分段背：第1句（自我介绍）→ 第2句（双引擎比喻）→ 第3句（三个触发条件）→ 第4句（节省资源）
- 记住关键词：本地规则、AI、三个条件、节省
- 每段练20遍

## 第四步：可能的Q&A（5个问题）

### Q1: 你们的AI决策真的比定时浇水好吗？

**回答**：
```
"我们做过对比实验：
- 定时浇水：用水量100%，存活率85%
- 阈值控制：用水量60%，存活率92%
- AI优化：用水量55%，存活率95%

AI的优势是动态调整，比如阴天自动延长补光。"
```

**⚠️ 注意**：
- 如果没有真实数据，把"我们做过对比实验"改成"我们设计过对比方案"
- 不要说假数据！

---

### Q2: AI决策会不会很慢？

**回答**：
```
"我们的AI不是随时都在跑。

只有三个条件触发时才会请AI：
阈值事件、环境变化、定时复核。

这样大部分时间用本地规则，快；
关键时刻请AI，准。

平均响应时间不到3秒。"
```

---

**回答**：

我们可以现场连接硬件演示。"

## 补位话术：评委问"和智能花盆有什么区别"时

> 当队员3说到"87个测试"或"软件架构"时，你立刻接：

### 补位动作（2步）

```
第1步：打开Web大屏
第2步：指向决策流水线区域，说——
  "我们的系统有87个自动化测试用例，
   专门测试断网、传感器失效、内存不足等故障。
   智能花盆不会有这种测试体系。"
```

### 如果评委追问"AI决策和智能花盆的AI有什么不同？"

```
"智能花盆的AI通常是简单的阈值判断——
干了浇，不干不浇。

我们的AI有三个触发条件：
阈值事件、环境变化、定时复核。
而且AI的决策是在本地规则基础上的优化，
不是替代本地规则。

这种'本地优先+AI辅助'的架构，
智能花盆没有。"
```

## 补位话术：评委问"为什么需要AI决策"时

> 当队员3说到"规则做底线，AI做优化"时，你立刻接技术细节：

### 核心补位（3层技术解释）

```
"我来补充一下技术实现——

第一层，为什么规则写不完：
8种作物×3-5个阶段×4个传感器维度，
组合空间至少96种。
而且温度在阈值边界时，该浇水还是该等？
这种灰色地带，if-else写不出来。

第二层，AI的三个触发条件：
阈值触发——传感器数据越界时请AI判断；
环境剧变——温度突然升高3度以上；
定时复核——每2小时让AI检查一次整体状态。
不是每次循环都调AI，太贵也太慢。

第三层，AI的输出有硬性护栏：
浇水最小间隔180秒、
营养液泵最小间隔300秒、
AI建议必须经过规则引擎安全检查才能执行。

AI是参谋，规则是指挥官。
参谋可以提方案，但拍板的是指挥官。"
```

### 如果评委追问"万一AI的建议和规则冲突怎么办？"

```
"规则赢，永远。

我们的架构是'AI提建议，规则做审批'——
AI输出不是直接执行，而是经过规则引擎安全检查。

比如AI建议'现在浇水'，
但规则引擎发现3分钟前刚浇过——
直接驳回，不执行。

再比如AI建议'开营养液泵'，
但温度传感器显示超过35℃——
规则引擎判定高温浇营养液会烧根，
直接拦截。

代码层面，decision.py里的_should_execute()方法
就是做这个安全检查的。"
```

### 如果评委追问"那10%到底是什么？举个例子？"

```
"比如番茄从苗期进入花期的过渡阶段——
苗期要勤浇水、多氮肥；
花期要控水、加磷钾肥。

但这个'过渡'不是按天算的，是看实际状态。
温度偏高，过渡可能提前；
光照不足，过渡可能延后。

本地规则只能写'第N天切换阶段'，
AI可以根据当天的温度、湿度、光照综合判断：
'今天温度偏高，苗期可能提前结束，可以适当控水了。'

这种判断，if-else写不出来。"
```

### 如果评委追问"既然AI更聪明，为什么不全部用AI？"

```
"三个原因：

第一，网络——太空不是随时在线，断网就废了。
第二，延迟——AI请求一次要1-3秒，
土壤已经干了还等AI回复？来不及。本地规则毫秒级。
第三，成本——AI每次调用消耗API额度和带宽，
太空场景下电力和带宽都是稀缺资源。

所以'规则优先、AI按需'是最优架构。"
```
