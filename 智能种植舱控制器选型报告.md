## 二、控制器选型对比分析

### 2.1 候选方案一览

| 方案 | 控制器 | 本地AI能力 | WiFi | 功耗 | 价格 | 推荐度 |
|------|--------|------------|------|------|------|--------|
| A | ESP32 普通版 | 弱 | 强 | 0.5W | ¥18 | **首选** |
| B | ESP32-S3 | 较弱 | 强 | 0.5W | ¥38 | 进阶 |
| C | Canaan K230 | 极强（离线） | 需外接 | 2-3W | ¥120 | 离线AI |
| D | 树莓派 4/5 | 最强 | 内置 | 5-15W | ¥450 | 展示用 |

### 2.2 选型结论

> **无摄像头需求 → 普通 ESP32（方案A）**  
> **有摄像头需求 / 未来扩展 → ESP32-S3（方案B）**

**理由：**
1. 普通 ESP32 有 **26 个 GPIO**，传感器 + 执行器共需 6-8 个引脚，绰绰有余
2. WiFi 性能稳定，适合向云端 AI 推送数据
3. 价格最低（¥18），适合课程批量采购
4. MicroPython 生态成熟，代码易维护

### 2.3 三款主控板详细对比

#### ESP32 普通版（推荐）

- **价格：** ¥18-25
- **处理器：** Xtensa LX6 单核 / 双核，240MHz
- **内存：** 520KB SRAM + 4MB Flash
- **GPIO：** 26 个（部分复用）
- **通信：** WiFi 802.11 b/g/n + Bluetooth 4.2
- **ADC：** 2 个 12 位 SAR ADC，最多 18 通道
- **UART：** 3 组（用 1 组接 CO2 传感器）
- **I2C：** 1 组（接 OLED 屏幕）
- **功耗：** 0.5W（正常），深度睡眠 < 0.01W
- **AI能力：** 端侧弱，需配合云端 AI

#### ESP32-S3（进阶选择）

- **相比普通版增加：**
  - USB-OTG 支持，可接摄像头模块
  - 双核 240MHz（更稳定）
  - 更强的 WiFi 性能（更远距离）
  - 更大内存，支持更多扩展

#### Canaan K230（离线AI场景）

- 内置神经网络加速单元（KPU）
- 可运行 TinyMaix 植物识别模型
- 完全离线，无需网络
- 适合偏远地区/地下室部署
- **缺点：** 无内置 WiFi，GPIO 需注意引脚定义

### 4.2 接线注意事项

#### 电源部分
```
12V/2A 电源适配器
    │
    ├──→ 12V 继电器（风扇/营养液泵）
    │         │
    │         └──→ 共地（GND）
    │
    └──→ AMS1117-5V 降压模块
              │
              ├──→ ESP32 Vin 引脚（5V供电）
              ├──→ 5V 继电器（水泵）
              ├──→ OLED 5V
              ├──→ DHT22 5V
              └──→ 传感器共地（GND）
```

#### 重要：共地必须连接
- ESP32 的 GND
- 所有继电器的 GND
- 所有传感器的 GND
- 电源适配器的负极

**如果不共地，继电器无法正常工作，水泵/风扇会失控。**

#### 继电器接线
```
ESP32 GPIO5 ──→ 继电器模块 IN
                 │
                 ├──→ VCC（5V）
                 └──→ GND
继电器 COM ──────→ 水泵正极
继电器 NO  ──────→ 12V电源正极
继电器 GND ──────→ 电源负极（与ESP32共地）
```

```python
import urequests
import ujson

def query_ai_decision(plant_type, soil_moisture, co2, temp, humidity):
    url = "https://open.bigmodel.cn/api/paas/v4/chat/completions"
    
    headers = {
        "Authorization": f"Bearer {API_KEY}",
        "Content-Type": "application/json"
    }
    
    plant_info = PLANT_DB[plant_type]
    
    system_prompt = f"""你是一个智能种植助手。
当前植物类型：{plant_type}
该植物的最适土壤湿度：{plant_info['soil_threshold']}%
该植物的最适CO2浓度：{plant_info['co2_threshold']}ppm

请根据以下传感器数据做出决策：
- 土壤湿度：{soil_moisture}%
- CO2浓度：{co2}ppm
- 环境温度：{temp}°C
- 环境湿度：{humidity}%

输出JSON格式（仅输出JSON，不要其他内容）：
{{
  "action": "water" | "nutrient" | "ventilate" | "idle",
  "duration_sec": 数字（执行时长，idle时为0）,
  "reason": "判断依据（10字以内）"
}}"""

    payload = {
        "model": "glm-4-flash",
        "messages": [{"role": "system", "content": system_prompt}],
        "temperature": 0.3
    }
    
    try:
        response = urequests.post(url, json=payload, headers=headers, timeout=3)
        result = ujson.loads(response.text)
        content = result['choices'][0]['message']['content']
        return ujson.loads(content)
    except:
        return None
```

### 5.3 本地规则兜底（断网备用）

```python
def local_fallback_decision(soil, co2, plant_info, last_nutrient, current_time):
    """本地备用决策逻辑（云端超时/失败时使用）"""
    
    soil_threshold = plant_info['soil_threshold']
    co2_threshold = plant_info['co2_threshold']
    water_sec = plant_info['water_sec']
    nutrient_sec = plant_info['nutrient_sec']
    ventilate_sec = plant_info['ventilate_sec']
    nutrient_interval = plant_info.get('nutrient_interval', 259200)  # 默认3天
    
    # 决策优先级
    # 1. 土壤极度干燥 -> 立即浇水
    if soil < soil_threshold - 15:
        return {"action": "water", "duration_sec": water_sec + 3, "reason": "土壤极度干燥"}
    
    # 2. 土壤干燥 -> 浇水
    if soil < soil_threshold:
        return {"action": "water", "duration_sec": water_sec, "reason": "土壤干燥"}
    
    # 3. CO2 过高 -> 换气
    if co2 > co2_threshold + 300:
        return {"action": "ventilate", "duration_sec": ventilate_sec + 15, "reason": "CO2严重超标"}
    
    if co2 > co2_threshold:
        return {"action": "ventilate", "duration_sec": ventilate_sec, "reason": "CO2偏高"}
    
    # 4. 需要补充营养液（定时）
    time_since_nutrient = current_time - last_nutrient
    if time_since_nutrient > nutrient_interval:
        if soil < soil_threshold + 15:
            return {"action": "nutrient", "duration_sec": nutrient_sec, "reason": "定时补充营养"}
    
    # 5. 一切正常
    return {"action": "idle", "duration_sec": 0, "reason": "状态正常"}
```

### 5.4 执行器控制函数

```python
from machine import Pin
import time

# 继电器初始化（低电平触发，继电器默认闭合=关闭）
WATER_PUMP_PIN   = Pin(5, Pin.OUT, value=1)  # GPIO5，水泵
NUTRIENT_PUMP_PIN = Pin(18, Pin.OUT, value=1) # GPIO18，营养液泵
FAN_PIN          = Pin(19, Pin.OUT, value=1)  # GPIO19，风扇

def execute_action(action, duration_sec):
    """执行指定动作，超时自动关闭"""
    
    if action == "idle":
        return
    
    action_map = {
        "water":      (WATER_PUMP_PIN,   "浇水"),
        "nutrient":   (NUTRIENT_PUMP_PIN, "营养液"),
        "ventilate":  (FAN_PIN,           "换气")
    }
    
    pin, name = action_map.get(action, (None, None))
    if not pin:
        return
    
    print(f"[执行] {name} {duration_sec}秒")
    update_oled(action, duration_sec)
    
    pin.value(0)           # 低电平触发：开
    time.sleep(duration_sec)
    pin.value(1)           # 关
    
    print(f"[完成] {name}完成")
```

```python
import machine
import time
import gc
import sys

# 本地模块
import config
import wifi_client
import sensors
import actuators
import display
import ai_client
import utils

# 全局状态
class SystemState:
    def __init__(self):
        self.wifi_connected = False
        self.soil_moisture = 0
        self.co2_ppm = 0
        self.temperature = 0.0
        self.humidity = 0.0
        self.plant_type = "生菜"
        self.days_since_planting = 0
        self.growth_stage = None
        self.last_action = "idle"
        self.last_action_time = 0
        self.last_nutrient_time = 0
        self.action_count = 0
        self.action_count_start = 0
        self.error_count = 0
        self.start_time = 0

state = SystemState()

def init_system():
    """系统初始化"""
    print("=" * 50)
    print("  太空农业种植舱控制系统 v1.0")
    print("=" * 50)
    
    state.start_time = time.time()
    state.action_count_start = time.time()
    
    # 按依赖顺序初始化：显示 → 传感器 → 执行器 → LED
    display.init()
    display.show_boot()
    sensors.init()
    actuators.init()
    utils.init_leds()
    
    # 连接WiFi
    state.wifi_connected = wifi_client.connect()
    
    # CO2传感器预热
    print("[系统] CO2传感器预热中 (30秒)...")
    display.show_text("CO2 Warming...", 20, 40)
    for i in range(30):
        time.sleep(1)
        if i % 5 == 0:
            display.show_text(f"CO2 Warming...{i+1}s", 20, 40)
    
    state.plant_type = sensors.read_plant_type()
    print("[植物] 当前类型:", state.plant_type)
    print("[系统] 初始化完成，开始主循环")
    return True

def read_all_sensors():
    """读取所有传感器，检测离线并降级"""
    soil = sensors.read_soil_moisture()
    co2 = sensors.read_co2()
    temp, hum = sensors.read_dht22()
    plant = sensors.read_plant_type()
    
    # 传感器离线检测
    sensor_failures = []
    if soil is None:
        sensor_failures.append("Soil")
        soil = 0  # 降级为0，触发安全浇水
    if co2 is None:
        sensor_failures.append("CO2")
        co2 = config.CO2_DANGER_HIGH  # 降级为高值，触发安全换气
    if temp is None or hum is None:
        sensor_failures.append("DHT22")
        temp = temp if temp is not None else 25.0
        hum = hum if hum is not None else 60.0
    
    if sensor_failures:
        fail_msg = "OFFLINE: " + ",".join(sensor_failures)
        print(f"[告警] 传感器离线: {fail_msg}")
        utils.set_led("red")
        display.show_error(fail_msg)
        time.sleep(2)
        state.error_count += 1
    
    state.soil_moisture = soil
    state.co2_ppm = co2
    state.temperature = temp
    state.humidity = hum
    state.plant_type = plant
    
    # 计算生长天数和当前阶段
    state.days_since_planting = config.calc_days_since_planting()
    plant_info = config.get_plant_info(plant)
    state.growth_stage = config.get_growth_stage(plant_info, state.days_since_planting)
    
    stage_name = state.growth_stage.get("stage", "unknown")
    print(f"[传感器] 土壤:{soil}% | CO2:{co2}ppm | 温:{temp}C | 湿:{hum}%")
    print(f"[生长] 第{state.days_since_planting}天 | 阶段:{stage_name}")
    return True

def make_decision():
    """AI决策 + 本地规则兜底"""
    plant_info = config.get_plant_info(state.plant_type)
    
    ai_result = None
    if state.wifi_connected:
        ai_result = ai_client.query_decision(
            plant_type=state.plant_type,
            soil_moisture=state.soil_moisture,
            co2=state.co2_ppm,
            temperature=state.temperature,
            humidity=state.humidity,
            plant_info=plant_info,
            days_since_planting=state.days_since_planting,
            growth_stage=state.growth_stage
        )
    
    if ai_result:
        return ai_result
    
    # 本地规则兜底
    return utils.local_fallback_decision(
        soil=state.soil_moisture,
        co2=state.co2_ppm,
        plant_info=plant_info,
        last_nutrient=state.last_nutrient_time,
        current_time=time.time()
    )

def execute_decision(decision):
    """执行决策"""
    action = decision.get('action', 'idle')
    duration = decision.get('duration_sec', 0)
    reason = decision.get('reason', '')
    
    if action == 'idle':
        actuators.all_off()
        state.last_action = 'idle'
        state.last_action_time = time.time()
        return
    
    print(f"[动作] 执行: {action} ({duration}秒) 原因: {reason}")
    utils.set_led("yellow")
    
    if action == "water":
        actuators.run_water_pump(duration)
    elif action == "nutrient":
        actuators.run_nutrient_pump(duration)
    elif action == "ventilate":
        actuators.run_fan(duration)
    
    state.last_action = action
    state.last_action_time = time.time()
    state.action_count += 1
    
    if action == "nutrient":
        state.last_nutrient_time = state.last_action_time
    
    utils.set_led("green")
    display.show_action(action, duration, reason)

def safety_check():
    """安全检查"""
    now = time.time()
    
    # 检查执行器实际硬件状态
    if actuators.is_any_running():
        print("[安全] 执行器运行中，跳过")
        return False
    
    # 动作间隔防抖
    if state.last_action != "idle":
        elapsed = now - state.last_action_time
        if elapsed < config.MIN_ACTION_INTERVAL:
            print(f"[安全] 动作间隔过短({elapsed:.0f}s)，跳过")
            return False
    
    # 每小时动作次数上限
    if now - state.action_count_start >= 3600:
        state.action_count_start = now
        state.action_count = 0
    
    if state.action_count > config.MAX_ACTIONS_PER_HOUR:
        print("[安全] 本小时动作次数超限，等待...")
        time.sleep(60)
        return False
    
    return True

def watch_dog():
    if state.error_count > config.MAX_ERRORS:
        print("[看门狗] 错误次数过多，系统重启")
        machine.reset()

def main_loop():
    interval = config.READ_INTERVAL
    last_read = 0
    
    while True:
        try:
            now = time.time()
            
            if now - last_read >= interval:
                last_read = now
                
                if not read_all_sensors():
                    continue
                
                if not safety_check():
                    continue
                
                decision = make_decision()
                execute_decision(decision)
                
                display.show_data(
                    soil=state.soil_moisture,
                    co2=state.co2_ppm,
                    temp=state.temperature,
                    hum=state.humidity,
                    plant=state.plant_type,
                    action=state.last_action
                )
                
                if not wifi_client.is_connected():
                    state.wifi_connected = wifi_client.connect()
                
                gc.collect()
            
            time.sleep(1)
            
        except KeyboardInterrupt:
            actuators.all_off()
            break
        except Exception as e:
            print("[错误] 主循环异常:", e)
            state.error_count += 1
            watch_dog()
            time.sleep(5)

def run():
    if init_system():
        main_loop()
    else:
        machine.reset()

if __name__ == "__main__":
    run()
```

## 六、AI 决策逻辑详解

### 6.1 决策规则设计原理

AI 决策基于四个核心维度：

#### 维度一：土壤湿度判断
```
土壤湿度 < 植物阈值 - 15%  → 立即浇水（干旱紧急，延长3秒）
土壤湿度 < 植物阈值        → 浇水（干旱正常）
土壤湿度 >= 植物阈值       → 检查CO2
```

#### 维度二：CO2浓度判断
```
CO2 > 植物阈值 + 300ppm   → 立即换气（严重超标，延长15秒）
CO2 > 植物阈值             → 换气（浓度偏高）
CO2 <= 植物阈值            → 检查营养液
```

#### 维度三：营养液补充（定时）
```
距上次营养液 > nutrient_interval 且 土壤不太干 → 营养液
```

#### 维度四：生长阶段动态调整
```
苗期(seedling)      → 少水、氮肥促根
营养生长期(vegetative) → 正常水、氮肥催长
开花期(flowering)   → 减水、磷钾肥促花
结果期(fruiting)    → 多水、钾肥膨果
采收期(harvesting)  → 正常水、复合肥维持
```

系统根据 `PLANTING_DATE` 自动计算种植天数，匹配当前生长阶段，并将阶段信息（推荐肥料、需水强度、养护要点）随传感器数据一并发送给 AI。

### 6.2 AI 提示词工程

```python
SYSTEM_PROMPT = """你是一个专业的太空农业种植助手。

背景：
- 太空种植舱使用水培系统，资源有限，必须高效管理
- 你的决策直接影响植物生存和宇航员食物供给
- 系统需要根据植物生长阶段动态调整养护策略

决策原则：
1. 生长阶段优先：不同阶段需求不同（苗期少水少肥、生长期多水氮肥、开花期减水补磷钾、结果期多水钾肥）
2. 安全第一：任何决策都不能导致系统过载或损坏
3. 资源高效：太空每一滴水都很珍贵，精准灌溉
4. 预防为主：发现趋势问题提前处理
5. 简洁执行：每次只执行一个主要动作

动作类型：
- water: 浇水（土壤干燥时）
- nutrient: 补充营养液（需要施肥时，AI应结合生长阶段建议肥料类型）
- ventilate: 换气（CO2浓度过高时）
- idle: 待机（所有指标正常时）

肥料类型说明（含在reason中告知用户）：
- N=氮肥（促叶生长，苗期/营养生长期使用）
- P=磷肥（促根促花，开花期使用）
- K=钾肥（膨果增甜，结果期使用）
- PK=磷钾肥（开花期控旺促花）
- NK=氮钾肥（结果期兼顾生长和产量）

输入数据包含：
- 当前植物类型和种植天数
- 当前生长阶段（seedling/vegetative/flowering/fruiting/harvesting）
- 该阶段推荐肥料类型和需水强度
- 土壤湿度、CO2浓度、温湿度等实时传感器数据
- 该植物的阈值参数

输出格式（严格JSON，不要其他内容）：
{
  "action": "water|nutrient|ventilate|idle",
  "duration_sec": 数字（动作执行时长，idle时为0）,
  "reason": "判断依据（含肥料类型建议，15字以内）"
}"""
```

## 十、常见问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 继电器不吸合 | 继电器是**高电平触发**，代码写反了 | 确认继电器型号，修改 GPIO.value(0/1) |
| 水泵一直转不停 | 没有设置超时，或超时时间太长 | 必须带 `time.sleep(duration)` 后关闭 |
| WiFi 频繁断开 | 信号弱 / 多设备干扰 | 加外置天线，或降低发送频率 |
| CO2 读数不准 | 未预热 / 未校准 | 开机等待 3 分钟，放户外校准 420ppm |
| 土壤传感器腐烂 | 电阻式传感器长期泡水 | 换电容式传感器 |
| 拨码开关植物反了 | PULL_UP 输入取反逻辑，ON=0, OFF=1 | 确认拨码方向：开关拨到"ON"侧=闭合接地=选中该位 |
| 传感器离线告警 | 传感器未接/接线松动/供电不足 | 检查接线，离线时系统自动降级为安全模式运行 |
| ESP32 重启 | 电源功率不足 | 换更大功率电源（12V/2A 以上） |
