## 二、控制器选型对比分析

### 2.1 候选方案一览

| 方案 | 控制器 | 本地AI能力 | WiFi | 功耗 | 价格 | 推荐度 |
|------|--------|------------|------|------|------|--------|
| A | ESP32 普通版 | 弱 | 强 | 0.5W | ¥18 | **首选** |
| B | ESP32-S3 | 较弱 | 强 | 0.5W | ¥38 | 进阶 |
| C | Canaan K230 | 极强（离线） | 需外接 | 2-3W | ¥120 | 离线AI |
| D | 树莓派 4/5 | 最强 | 内置 | 5-15W | ¥450 | 展示用 |

### 2.2 选型结论

> **无摄像头需求 → 普通 ESP32（方案A）**  
> **有摄像头需求 / 未来扩展 → ESP32-S3（方案B）**

**理由：**
1. 普通 ESP32 有 **26 个 GPIO**，传感器 + 执行器共需 6-8 个引脚，绰绰有余
2. WiFi 性能稳定，适合向云端 AI 推送数据
3. 价格最低（¥18），适合课程批量采购
4. MicroPython 生态成熟，代码易维护

### 2.3 三款主控板详细对比

#### ESP32 普通版（推荐）

- **价格：** ¥18-25
- **处理器：** Xtensa LX6 单核 / 双核，240MHz
- **内存：** 520KB SRAM + 4MB Flash
- **GPIO：** 26 个（部分复用）
- **通信：** WiFi 802.11 b/g/n + Bluetooth 4.2
- **ADC：** 2 个 12 位 SAR ADC，最多 18 通道
- **UART：** 3 组（用 1 组接 CO2 传感器）
- **I2C：** 1 组（接 OLED 屏幕）
- **功耗：** 0.5W（正常），深度睡眠 < 0.01W
- **AI能力：** 端侧弱，需配合云端 AI

#### ESP32-S3（进阶选择）

- **相比普通版增加：**
  - USB-OTG 支持，可接摄像头模块
  - 双核 240MHz（更稳定）
  - 更强的 WiFi 性能（更远距离）
  - 更大内存，支持更多扩展

#### Canaan K230（离线AI场景）

- 内置神经网络加速单元（KPU）
- 可运行 TinyMaix 植物识别模型
- 完全离线，无需网络
- 适合偏远地区/地下室部署
- **缺点：** 无内置 WiFi，GPIO 需注意引脚定义

### 4.2 接线注意事项

#### 电源部分
```
12V/2A 电源适配器
    │
    ├──→ 12V 继电器（风扇/营养液泵）
    │         │
    │         └──→ 共地（GND）
    │
    └──→ AMS1117-5V 降压模块
              │
              ├──→ ESP32 Vin 引脚（5V供电）
              ├──→ 5V 继电器（水泵）
              ├──→ OLED 5V
              ├──→ DHT22 5V
              └──→ 传感器共地（GND）
```

#### 重要：共地必须连接
- ESP32 的 GND
- 所有继电器的 GND
- 所有传感器的 GND
- 电源适配器的负极

**如果不共地，继电器无法正常工作，水泵/风扇会失控。**

#### 继电器接线
```
ESP32 GPIO5 ──→ 继电器模块 IN
                 │
                 ├──→ VCC（5V）
                 └──→ GND
继电器 COM ──────→ 水泵正极
继电器 NO  ──────→ 12V电源正极
继电器 GND ──────→ 电源负极（与ESP32共地）
```

```python
import urequests
import ujson

def query_ai_decision(plant_type, soil_moisture, co2, temp, humidity):
    url = "https://open.bigmodel.cn/api/paas/v4/chat/completions"
    
    headers = {
        "Authorization": f"Bearer {API_KEY}",
        "Content-Type": "application/json"
    }
    
    plant_info = PLANT_DB[plant_type]
    
    system_prompt = f"""你是一个智能种植助手。
当前植物类型：{plant_type}
该植物的最适土壤湿度：{plant_info['soil_threshold']}%
该植物的最适CO2浓度：{plant_info['co2_threshold']}ppm

请根据以下传感器数据做出决策：
- 土壤湿度：{soil_moisture}%
- CO2浓度：{co2}ppm
- 环境温度：{temp}°C
- 环境湿度：{humidity}%

输出JSON格式（仅输出JSON，不要其他内容）：
{{
  "action": "water" | "nutrient" | "ventilate" | "idle",
  "duration_sec": 数字（执行时长，idle时为0）,
  "reason": "判断依据（10字以内）"
}}"""

    payload = {
        "model": "glm-4-flash",
        "messages": [{"role": "system", "content": system_prompt}],
        "temperature": 0.3
    }
    
    try:
        response = urequests.post(url, json=payload, headers=headers, timeout=3)
        result = ujson.loads(response.text)
        content = result['choices'][0]['message']['content']
        return ujson.loads(content)
    except:
        return None
```

### 5.3 本地规则兜底（断网备用）

```python
def local_fallback_decision(soil_moisture, co2, plant_info):
    """本地备用决策逻辑（云端超时/失败时使用）"""
    
    # 优先级：土壤干燥 > CO2过高 > 都需要
    if soil_moisture < plant_info['soil_threshold']:
        return {"action": "water", "duration_sec": plant_info['water_sec'], "reason": "土壤干燥"}
    
    if co2 > plant_info['co2_threshold']:
        return {"action": "ventilate", "duration_sec": plant_info['ventilate_sec'], "reason": "CO2过高"}
    
    if soil_moisture < plant_info['soil_threshold'] + 10:
        return {"action": "nutrient", "duration_sec": plant_info['nutrient_sec'], "reason": "需补充营养"}
    
    return {"action": "idle", "duration_sec": 0, "reason": "状态正常"}
```

### 5.4 执行器控制函数

```python
from machine import Pin
import time

# 继电器初始化（低电平触发，继电器默认闭合=关闭）
WATER_PUMP_PIN   = Pin(5, Pin.OUT, value=1)  # GPIO5，水泵
NUTRIENT_PUMP_PIN = Pin(18, Pin.OUT, value=1) # GPIO18，营养液泵
FAN_PIN          = Pin(19, Pin.OUT, value=1)  # GPIO19，风扇

def execute_action(action, duration_sec):
    """执行指定动作，超时自动关闭"""
    
    if action == "idle":
        return
    
    action_map = {
        "water":      (WATER_PUMP_PIN,   "浇水"),
        "nutrient":   (NUTRIENT_PUMP_PIN, "营养液"),
        "ventilate":  (FAN_PIN,           "换气")
    }
    
    pin, name = action_map.get(action, (None, None))
    if not pin:
        return
    
    print(f"[执行] {name} {duration_sec}秒")
    update_oled(action, duration_sec)
    
    pin.value(0)           # 低电平触发：开
    time.sleep(duration_sec)
    pin.value(1)           # 关
    
    print(f"[完成] {name}完成")
```

### 5.5 完整主循环

```python
import machine
import time
import network
from machine import Pin, UART, I2C, ADC
import ssd1306
import dht
import urequests
import ujson

# ============ 初始化 ============

# WiFi 连接
def connect_wifi():
    wlan = network.WLAN(network.STA_IF)
    wlan.active(True)
    wlan.connect("你的WiFi名称", "你的WiFi密码")
    
    timeout = 30
    while not wlan.isconnected() and timeout > 0:
        time.sleep(1)
        timeout -= 1
    
    if wlan.isconnected():
        print("[WiFi] 已连接:", wlan.ifconfig())
    else:
        print("[WiFi] 连接失败，将使用本地规则")
    return wlan.isconnected()

# OLED 初始化
i2c = I2C(0, scl=Pin(22), sda=Pin(21), freq=400000)
oled = ssd1306.SSD1306_I2C(128, 64, i2c)

# UART（CO2传感器）
uart = UART(1, baudrate=9600, tx=16, rx=17)

# ADC（土壤湿度）
soil_adc = ADC(Pin(34))
soil_adc.atten(ADC.ATTN_11DB)  # 0-3.3V

# DHT22
# dht22 = dht.DHT22(Pin(4))  # 注：ESP32 MicroPython可能需要用特殊驱动

# ============ 传感器读取 ============

def read_soil_moisture():
    """读取土壤湿度（返回百分比）"""
    raw = soil_adc.read()
    # 干土≈4095，湿土≈1500
    moisture = int((4095 - raw) / (4095 - 1500) * 100)
    return max(0, min(100, moisture))

def read_co2():
    """读取CO2浓度（ppm）"""
    uart.write(b'\xff\x87\x00\x00\x00\x00\x87')  # 读取命令
    time.sleep(0.1)
    if uart.any():
        data = uart.read()
        if len(data) >= 9:
            return (data[2] << 8) + data[3]
    return None

def read_plant_type():
    """读取拨码开关，判断植物类型"""
    dip1 = Pin(13, Pin.IN).value()
    dip2 = Pin(12, Pin.IN).value()
    dip3 = Pin(14, Pin.IN).value()
    index = (dip1 << 2) | (dip2 << 1) | dip3
    plants = ["生菜","小白菜","菠菜","韭菜","番茄","辣椒","黄瓜","茄子"]
    return plants[index]

# ============ OLED 显示 ============

def update_oled(action, duration):
    oled.fill(0)
    soil = read_soil_moisture()
    co2 = read_co2()
    
    oled.text("太空种植舱", 20, 0)
    oled.text(f"植物: {read_plant_type()}", 0, 16)
    oled.text(f"土壤: {soil}%", 0, 26)
    oled.text(f"CO2:  {co2}ppm", 0, 36)
    
    action_names = {"water": "浇水", "nutrient": "营养液", "ventilate": "换气", "idle": "待机"}
    oled.text(f"动作: {action_names.get(action, '未知')}", 0, 46)
    oled.show()

# ============ 主循环 ============

def main():
    connect_wifi()
    
    print("[系统] 启动，预热CO2传感器30秒...")
    oled.fill(0)
    oled.text("太空种植舱", 20, 20)
    oled.text("CO2预热中...", 20, 35)
    oled.show()
    time.sleep(30)  # CO2传感器预热
    
    interval = 300  # 5分钟执行一次
    
    while True:
        try:
            soil = read_soil_moisture()
            co2 = read_co2()
            plant_type = read_plant_type()
            plant_info = PLANT_DB[plant_type]
            
            # 尝试云端AI决策
            ai_result = None
            if wlan.isconnected():
                ai_result = query_ai_decision(plant_type, soil, co2, 25, 60)
            
            # 决策（云端失败则用本地规则）
            if ai_result:
                decision = ai_result
            else:
                decision = local_fallback_decision(soil, co2, plant_info)
            
            # 执行动作
            execute_action(decision['action'], decision['duration_sec'])
            
            # 显示状态
            update_oled(decision['action'], decision['duration_sec'])
            
        except Exception as e:
            print("[错误]", e)
            machine.reset()  # 异常时重启
        
        time.sleep(interval)

main()
```

## 六、AI 决策逻辑详解

### 6.1 决策规则设计原理

AI 决策基于三个核心维度：

#### 维度一：土壤湿度判断
```
土壤湿度 < 植物阈值 - 10%  → 立即浇水（干旱紧急）
土壤湿度 < 植物阈值        → 浇水（干旱正常）
土壤湿度 >= 植物阈值       → 检查CO2
```

#### 维度二：CO2浓度判断
```
CO2 > 植物阈值 + 300ppm   → 立即换气（严重缺氧/过浓）
CO2 > 植物阈值             → 换气（浓度偏高）
CO2 <= 植物阈值            → 检查营养液
```

#### 维度三：营养液补充（每周1-2次）
```
上次浇营养液时间 > 3天 且 土壤湿度偏低 → 营养液
```

## 十、常见问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 继电器不吸合 | 继电器是**高电平触发**，代码写反了 | 确认继电器型号，修改 GPIO.value(0/1) |
| 水泵一直转不停 | 没有设置超时，或超时时间太长 | 必须带 `time.sleep(duration)` 后关闭 |
| WiFi 频繁断开 | 信号弱 / 多设备干扰 | 加外置天线，或降低发送频率 |
| CO2 读数不准 | 未预热 / 未校准 | 开机等待 3 分钟，放户外校准 420ppm |
| 土壤传感器腐烂 | 电阻式传感器长期泡水 | 换电容式传感器 |
| ESP32 重启 | 电源功率不足 | 换更大功率电源（12V/2A 以上） |
