## 二、控制器选型对比分析

### 2.1 候选方案一览

| 方案 | 控制器       | 本地AI能力   | WiFi   | 功耗  | 价格 | 推荐度   |
| ---- | ------------ | ------------ | ------ | ----- | ---- | -------- |
| A    | ESP32 普通版 | 弱           | 强     | 0.5W  | ¥18  | **首选** |
| B    | ESP32-S3     | 较弱         | 强     | 0.5W  | ¥38  | 进阶     |
| C    | Canaan K230  | 极强（离线） | 需外接 | 2-3W  | ¥120 | 离线AI   |
| D    | 树莓派 4/5   | 最强         | 内置   | 5-15W | ¥450 | 展示用   |

### 2.2 选型结论

> **无摄像头需求 → 普通 ESP32（方案A）**  
> **有摄像头需求 / 未来扩展 → ESP32-S3（方案B）**

**理由：**

1. 普通 ESP32 有 **26 个 GPIO**，传感器 + 执行器共需 6-8 个引脚，绰绰有余
2. WiFi 性能稳定，适合向云端 AI 推送数据
3. 价格最低（¥18），适合课程批量采购
4. MicroPython 生态成熟，代码易维护

### 2.3 三款主控板详细对比

#### ESP32 普通版（推荐）

- **价格：** ¥18-25
- **处理器：** Xtensa LX6 单核 / 双核，240MHz
- **内存：** 520KB SRAM + 4MB Flash
- **GPIO：** 26 个（部分复用）
- **通信：** WiFi 802.11 b/g/n + Bluetooth 4.2
- **ADC：** 2 个 12 位 SAR ADC，最多 18 通道
- **I2C：** 1 组（接 OLED 屏幕）
- **功耗：** 0.5W（正常），深度睡眠 < 0.01W
- **AI能力：** 端侧弱，需配合云端 AI（支持直连或代理中转）

#### ESP32-S3（进阶选择）

- **相比普通版增加：**
  - USB-OTG 支持，可接摄像头模块
  - 双核 240MHz（更稳定）
  - 更强的 WiFi 性能（更远距离）
  - 更大内存，支持更多扩展

#### Canaan K230（离线AI场景）

- 内置神经网络加速单元（KPU）
- 可运行 TinyMaix 植物识别模型
- 完全离线，无需网络
- 适合偏远地区/地下室部署
- **缺点：** 无内置 WiFi，GPIO 需注意引脚定义

### 4.2 接线注意事项

#### 电源部分

```
12V/2A 电源适配器
    │
    ├──→ 12V 继电器（营养液泵）
    │         │
    │         └──→ 共地（GND）
    │
    └──→ AMS1117-5V 降压模块
              │
              ├──→ ESP32 Vin 引脚（5V供电）
              ├──→ 5V 继电器（水泵）
              ├──→ OLED 5V
              ├──→ DHT11 3.3V/5V
              ├──→ 光敏模块 VCC
              └──→ 传感器共地（GND）
```

#### 重要：共地必须连接

- ESP32 的 GND
- 所有继电器的 GND
- 所有传感器的 GND
- 电源适配器的负极

**如果不共地，继电器无法正常工作，水泵会失控。**

#### 继电器接线

```
ESP32 GPIO5 ──→ 继电器模块 IN
                 │
                 ├──→ VCC（5V）
                 └──→ GND
继电器 COM ──────→ 水泵正极
继电器 NO  ──────→ 12V电源正极
继电器 GND ──────→ 电源负极（与ESP32共地）
```

#### 光敏模块接线

```
HS-S20L-B 光敏模块
    VCC ──→ 3.3V
    GND ──→ GND
    AO  ──→ GPIO32（模拟量输出）
```

### 5.2 AI 决策 API 设计

#### 请求格式（支持直连或代理中转）

ESP32 内存有限（520KB SRAM），TLS 握手需要大量堆内存。系统提供两种模式：

- **代理中转模式（推荐）：** ESP32 → HTTP 代理服务器 → DeepSeek API。代理使用普通 HTTP，无需 TLS，内存占用低且稳定。
- **直连模式：** ESP32 → HTTPS → DeepSeek API。需在 AI 请求前释放 OLED 帧缓冲，腾出内存给 TLS 握手。

```python
Actions:
- water: if soil dry
- nutrient: if fert needed

        result = ujson.loads(response.text)

    except:
        return None
```

### 5.3 本地规则兜底（断网备用）

```python
def local_fallback_decision(soil, plant_info, last_nutrient, current_time):
    """本地备用决策逻辑（云端超时/失败时使用）"""

    soil_threshold = plant_info['soil_threshold']
    water_sec = plant_info['water_sec']
    nutrient_sec = plant_info['nutrient_sec']
    nutrient_interval = plant_info.get('nutrient_interval', 259200)  # 默认3天

    # 决策优先级
    # 1. 土壤极度干燥 -> 立即浇水
    if soil < soil_threshold - 15:
        return {"action": "water", "duration_sec": water_sec + 3, "reason": "土壤极度干燥"}

    # 2. 土壤干燥 -> 浇水
    if soil < soil_threshold:
        return {"action": "water", "duration_sec": water_sec, "reason": "土壤干燥"}

    # 3. 需要补充营养液（定时）
    time_since_nutrient = current_time - last_nutrient
    if time_since_nutrient > nutrient_interval:
        if soil < soil_threshold + 15:
            return {"action": "nutrient", "duration_sec": nutrient_sec, "reason": "定时补充营养"}

    # 4. 一切正常
    return {"action": "idle", "duration_sec": 0, "reason": "状态正常"}
```

### 5.4 执行器控制函数

```python
from machine import Pin
import time
import config

# 继电器初始化（低电平触发，默认关闭=value=1）
_water_relay = Pin(config.RELAY_WATER_PIN, Pin.OUT, value=1)
_nutrient_relay = Pin(config.RELAY_NUTRIENT_PIN, Pin.OUT, value=1)

def run_water_pump(duration_sec):
    """运行水泵，带安全时长限制和分段执行"""
    duration = min(duration_sec, config.PUMP_MAX_RUN_SEC)
    if duration <= 0:
        return False

    _water_relay.value(0)  # 低电平触发：开
    remaining = duration
    while remaining > 0:
        time.sleep(min(remaining, 1))  # 每秒检测一次中断
        remaining -= 1
    _water_relay.value(1)  # 关
    return True

# run_nutrient_pump() 结构相同，控制营养液泵继电器
```

### 5.5 完整主循环（模块化版本）

固件采用模块化设计，主文件 `main.py` 如下：

```python
import machine
import time
import gc
import sys

# 本地模块
import config
import wifi_client
import sensors
import actuators
import utils

# 全局状态
class SystemState:
    def __init__(self):
        self.wifi_connected = False
        self.soil_moisture = 0
        self.light_level = 0
        self.temperature = 0.0
        self.humidity = 0.0
        self.plant_type = "生菜"
        self.days_since_planting = 0
        self.growth_stage = None
        self.last_action = "idle"
        self.last_action_time = 0
        self.last_nutrient_time = 0
        self.action_count = 0
        self.action_count_start = 0
        self.error_count = 0
        self.start_time = 0

state = SystemState()

def init_system():
    """系统初始化"""
    print("=" * 50)
    print("  Space Agriculture Growth Chamber System v1.0")
    print("=" * 50)

    state.start_time = time.time()
    state.action_count_start = time.time()

    # 按依赖顺序初始化：显示 → 传感器 → 执行器 → LED
    _init_display()
    _display().show_boot()
    sensors.init()
    actuators.init()
    utils.init_leds()

    # 连接WiFi
    state.wifi_connected = wifi_client.connect()

    # 首次完整读取传感器
    read_all_sensors()
    print("[Plant] Current type:", state.plant_type)
    _display().show_idle(state.soil_moisture, state.light_level,
                         state.plant_type, state.temperature, state.humidity)
    print("[System] Initialization complete, starting main loop")
    return True

def read_all_sensors():
    """读取所有传感器，检测离线并降级"""
    soil = sensors.read_soil_moisture()
    light = sensors.read_light_level()
    temp, hum = sensors.read_dht22()
    plant = sensors.read_plant_type()

    # 传感器离线检测
    sensor_failures = []
    if soil is None:
        sensor_failures.append("Soil")
        soil = 0  # 降级为0，触发安全浇水
    if light is None:
        sensor_failures.append("Light")
        light = 0
    if temp is None or hum is None:
        sensor_failures.append("DHT")
        temp = temp if temp is not None else 25.0
        hum = hum if hum is not None else 60.0

    if sensor_failures:
        fail_msg = "OFFLINE: " + ",".join(sensor_failures)
        print(f"[Alert] Sensor offline: {fail_msg}")
        utils.set_led("red")
        _display().show_error(fail_msg)
        time.sleep(2)
        state.error_count += 1

    state.soil_moisture = soil
    state.light_level = light
    state.temperature = temp
    state.humidity = hum
    state.plant_type = plant

    # 计算生长天数和当前阶段
    state.days_since_planting = config.calc_days_since_planting()
    plant_info = config.get_plant_info(plant)
    state.growth_stage = config.get_growth_stage(plant_info, state.days_since_planting)

    stage_name = state.growth_stage.get("stage", "unknown")
    print(f"[Sensor] Soil:{soil}% | Light:{light}% | Temp:{temp}C | Hum:{hum}%")
    print(f"[Growth] Day {state.days_since_planting} | Stage: {stage_name}")

    # 成功读取，重置连续错误计数
    state.error_count = 0
    return True

def make_decision():
    """AI决策 + 本地规则兜底"""
    plant_info = config.get_plant_info(state.plant_type)
    ai_plant_info = {"soil_threshold": plant_info["soil_threshold"]}

    # 尝试云端AI
    ai_result = None
    if state.wifi_connected:
        use_proxy = bool(getattr(config, "AI_PROXY_URL", ""))
        if not use_proxy:
            _release_display()  # 直连模式需释放OLED内存给TLS
        gc.collect()
        free_mem = gc.mem_free()
        min_ai_mem = getattr(config, "AI_MIN_FREE_MEM", 110000)
        if use_proxy or free_mem >= min_ai_mem:
            import ai_client
            ai_result = ai_client.query_decision(
                plant_type=state.plant_type,
                soil_moisture=state.soil_moisture,
                light_level=state.light_level,
                temperature=state.temperature,
                humidity=state.humidity,
                plant_info=ai_plant_info,
                days_since_planting=state.days_since_planting,
                growth_stage=state.growth_stage
            )
        else:
            print(f"[AI] Low memory ({free_mem} bytes), using local rules")

    if ai_result:
        return ai_result

    # 本地规则兜底
    return utils.local_fallback_decision(
        soil=state.soil_moisture,
        plant_info=plant_info,
        last_nutrient=state.last_nutrient_time,
        current_time=time.time()
    )

def execute_decision(decision):
    """执行决策"""
    action = decision.get('action', 'idle')
    duration = min(decision.get('duration_sec', 0), config.PUMP_MAX_RUN_SEC)
    reason = decision.get('reason', '')

    if action == 'idle':
        actuators.all_off()
        _display().show_idle(state.soil_moisture, state.light_level,
                             state.plant_type, state.temperature, state.humidity)
        state.last_action = 'idle'
        state.last_action_time = time.time()
        return

    print(f"[Action] Executing: {action} ({duration}s) Reason: {reason}")
    utils.set_led("yellow")

    if action == "water":
        actuators.run_water_pump(duration)
    elif action == "nutrient":
        actuators.run_nutrient_pump(duration)

    state.last_action = action
    state.last_action_time = time.time()
    state.action_count += 1

    if action == "nutrient":
        state.last_nutrient_time = state.last_action_time

    utils.set_led("green")
    _display().show_action(action, duration, reason)

def safety_check():
    """安全检查"""
    now = time.time()

    if actuators.is_any_running():
        print("[Safety] Actuator running, skipped")
        return False

    if state.last_action != "idle":
        elapsed = now - state.last_action_time
        if elapsed < config.MIN_ACTION_INTERVAL:
            print(f"[Safety] Action interval too short ({elapsed:.0f}s), skipped")
            return False

    if now - state.action_count_start >= 3600:
        state.action_count_start = now
        state.action_count = 0

    if state.action_count >= config.MAX_ACTIONS_PER_HOUR:
        print("[Safety] Hourly action limit exceeded, waiting...")
        time.sleep(60)
        return False

    return True

def watch_dog():
    if state.error_count > config.MAX_ERRORS:
        print("[Watchdog] Too many errors, system restarting")
        machine.reset()

def main_loop():
    interval = config.READ_INTERVAL
    last_read = 0

    while True:
        try:
            now = time.time()

            if now - last_read >= interval:
                last_read = now

                if not read_all_sensors():
                    continue

                if not safety_check():
                    continue

                decision = make_decision()
                execute_decision(decision)

                _display().show_data(
                    soil=state.soil_moisture,
                    light=state.light_level,
                    temp=state.temperature,
                    hum=state.humidity,
                    plant=state.plant_type,
                    action=state.last_action
                )

                if not wifi_client.is_connected():
                    state.wifi_connected = wifi_client.smart_connect()

                gc.collect()

            time.sleep(1)

        except KeyboardInterrupt:
            actuators.all_off()
            break
        except Exception as e:
            print("[Error] Main loop exception:", e)
            state.error_count += 1
            watch_dog()
            time.sleep(5)

def run():
    if init_system():
        main_loop()
    else:
        machine.reset()

if __name__ == "__main__":
    run()
```

## 六、AI 决策逻辑详解

### 6.1 决策规则设计原理

AI 决策基于三个核心维度：

#### 维度一：土壤湿度判断

```
土壤湿度 < 植物阈值 - 15%  → 立即浇水（干旱紧急，延长3秒）
土壤湿度 < 植物阈值        → 浇水（干旱正常）
土壤湿度 >= 植物阈值       → 检查营养液
```

#### 维度二：营养液补充（定时）

```
距上次营养液 > nutrient_interval 且 土壤不太干 → 营养液
```

#### 维度三：生长阶段动态调整

```
苗期(seedling)      → 少水、氮肥促根
营养生长期(vegetative) → 正常水、氮肥催长
开花期(flowering)   → 减水、磷钾肥促花
结果期(fruiting)    → 多水、钾肥膨果
采收期(harvesting)  → 正常水、复合肥维持
```

系统根据 `PLANTING_DATE` 自动计算种植天数，匹配当前生长阶段，并将阶段信息（推荐肥料、需水强度、养护要点）随传感器数据（土壤湿度、光照、温湿度）一并发送给 AI。

### 6.2 AI 提示词工程

```python
SYSTEM_PROMPT = """You are a space agriculture AI.
Rules:
1. Stage needs: Seedling=low water/fert; Veg=high water/N; Bloom=low water, high P/K; Fruit=high water/K.
2. Safety first. Avoid system overload.
3. Save water.
4. One action at a time.
Actions:
- water: if soil dry
- nutrient: if fert needed
- idle: if normal
Fertilizers: N, P, K, PK, NK.
Output strict JSON:
{"action":"water|nutrient|idle","duration_sec":int,"reason":"short reason"}"""
```

> **提示词设计说明**：使用英文提示词减少 token 消耗（中文 token 消耗约为英文的 2-3 倍），同时避免 MicroPython `urequests` 对中文 Content-Length 计算的 bug。用户输入中植物名称和阈值参数保持中文，AI 仍能正确理解。

### 6.3 AI 代理中转架构

```
ESP32 ──HTTP──→ 代理服务器 ──HTTPS──→ DeepSeek API
         (无TLS)      (有TLS)
```

**为什么需要代理？**

- ESP32 的 RAM 仅 520KB，TLS 握手需要约 40-60KB 连续堆内存
- 直连模式下，AI 请求前必须释放 OLED 帧缓冲（~1KB）和 ssd1306 模块
- 代理模式使用普通 HTTP，无需 TLS，内存压力小，运行更稳定
- 代理服务器还可在云端做日志记录、请求频率限制等

**配置方式：**

```python
# config.py
AI_PROXY_URL = "http://your-proxy-server:8787/decision"  # 为空则 ESP32 直连
AI_PROXY_TOKEN = "your-secret-token"  # 代理访问令牌
```

## 十、常见问题与解决方案

| 问题             | 原因                               | 解决方案                                       |
| ---------------- | ---------------------------------- | ---------------------------------------------- |
| 继电器不吸合     | 继电器是**高电平触发**，代码写反了 | 确认继电器型号，修改 GPIO.value(0/1)           |
| 水泵一直转不停   | 没有设置超时，或超时时间太长       | 必须带 `time.sleep(duration)` 后关闭           |
| WiFi 频繁断开    | 信号弱 / 多设备干扰                | 加外置天线，或降低发送频率                     |
| AI 请求失败      | ESP32 内存不足，TLS 握手失败       | 使用代理中转模式，或释放 OLED 内存             |
| 土壤传感器腐烂   | 电阻式传感器长期泡水               | 换电容式传感器                                 |
| 拨码开关植物反了 | PULL_UP 输入取反逻辑，ON=0, OFF=1  | 确认拨码方向：开关拨到"ON"侧=闭合接地=选中该位 |
| 传感器离线告警   | 传感器未接/接线松动/供电不足       | 检查接线，离线时系统自动降级为安全模式运行     |
| ESP32 重启       | 电源功率不足                       | 换更大功率电源（12V/2A 以上）                  |
| DHT11 读取失败   | 读取间隔小于1秒                    | 代码已内置3次重试，每次间隔800ms               |
| OLED 显示花屏    | I2C 通信受干扰                     | 检查接线，确保 SDA/SCL 有上拉电阻              |
