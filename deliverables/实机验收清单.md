# 实机验收清单

用于比赛、答辩或课堂展示前的最后一轮硬件在环检查。建议每次更换 ESP32、传感器、WiFi、服务器地址或水泵供电后重新跑一遍。

## 1. 准备

- ESP32 已烧录 MicroPython，串口可通过 `mpremote` 连接。
- `esp32_firmware/config.py` 已由 `config.py.example` 复制并填好 WiFi、Dashboard、AI 代理地址。
- 已上传 `plants.json`、`main.py` 以及所有 `*_runtime.py` 模块。
- 水泵和营养液泵先断开真实水路，第一次验收只观察继电器和电机动作。
- Dashboard 服务已启动，浏览器能打开实时大屏。

## 2. 冷启动检查

| 检查项 | 期望结果 | 记录 |
| --- | --- | --- |
| ESP32 上电 | 串口打印启动横幅，无异常重启 |  |
| OLED 启动页 | 显示 boot、WiFi 状态、System Ready |  |
| WiFi 连接 | 串口打印 IP，OLED 显示 WiFi OK |  |
| WiFi 失败场景 | 系统不崩溃，进入本地规则模式 |  |
| 首次采样 | OLED 和串口出现真实或模拟传感器值 |  |

## 3. 传感器检查

| 检查项 | 操作 | 期望结果 | 记录 |
| --- | --- | --- | --- |
| 土壤湿度 | 干/湿状态切换 | 湿度百分比有明显变化 |  |
| 光照强度 | 遮挡/照射光敏模块 | 光照百分比有明显变化 |  |
| 温湿度 | 读取 DHT | 温度、湿度不是默认错误值 |  |
| 作物拨码 | 切换拨码后重启 | 作物类型变化，阈值随作物变化 |  |
| 传感器离线 | 拔掉一个传感器或模拟 None | OLED 显示 OFFLINE，系统降级但不中断 |  |

## 4. 决策与执行检查

| 检查项 | 操作 | 期望结果 | 记录 |
| --- | --- | --- | --- |
| 本地规则浇水 | 将土壤湿度降到阈值以下 | 决策为 water，水泵动作时长不超过上限 |  |
| 待机状态 | 环境值恢复正常 | 决策为 idle，执行器全部关闭 |  |
| 动作间隔保护 | 短时间内连续触发 | 第二次动作被安全检查拦截 |  |
| 每小时动作上限 | 模拟动作计数达到上限 | 系统打印限流信息，不继续执行 |  |
| 营养液动作 | 模拟营养间隔到期 | nutrient 动作可执行，并记录最后施肥时间 |  |

## 5. AI 与网络检查

| 检查项 | 操作 | 期望结果 | 记录 |
| --- | --- | --- | --- |
| Dashboard 上报 | 保持 ESP32 在线 | 大屏采样次数、传感器值持续更新 |  |
| AI 代理可用 | 设置 `AI_PROXY_URL` | 阈值事件时允许云端决策 |  |
| AI 不可用 | 断开代理或清空密钥 | 系统回退本地规则，不影响浇水保护 |  |
| WiFi 断线重连 | 临时断开网络 | 串口打印重连信息，恢复后继续上报 |  |
| 内存回收 | 连续运行 30 分钟 | 无频繁重启，`gc.mem_free()` 不持续跌破安全线 |  |

## 6. 展示大屏检查

| 检查项 | 期望结果 | 记录 |
| --- | --- | --- |
| 顶部状态 | 网络、智能决策、模式、运行时间、种植天数显示合理 |  |
| 传感器卡片 | 异常传感器有醒目状态，正常传感器易读 |  |
| 趋势图 | 有时间轴和数值参照，曲线不互相误导 |  |
| 当前决策 | 能看出当前动作、原因、上次触发和连续待机次数 |  |
| 执行记录 | 记录不过度重复，最近动作可追踪 |  |

## 7. 通过标准

- 连续运行 30 分钟无异常重启。
- 传感器离线、WiFi 断线、AI 失败均能降级运行。
- 水泵和营养液泵不会超过配置的安全时长和频次。
- Dashboard 能展示从采样到决策再到执行的完整闭环。
- 串口日志、OLED、大屏三处信息相互一致。
