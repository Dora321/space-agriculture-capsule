# 交付物索引

这个目录存放面向比赛、展示和评审的材料。为了避免文件越堆越乱，建议按下面的用途理解和维护。

## 推荐入口

| 文件 | 用途 |
| --- | --- |
| `contest-demo-dashboard.html` | 当前实时/演示大屏，部署在 Dashboard 服务首页 |
| `realtime-dashboard-guide.md` | 大屏本地运行、云服务器端口和 ESP32 上报配置说明 |
| `display-redesign-proposal-2026-05-16.md` | 大屏改版方案和设计背景 |

## 本地待确认交付物

| 文件 | 用途 |
| --- | --- |
| `评委展示方案.md` | 面向评委讲解的展示流程和话术 |
| `KT板设计文档.md` | KT 板内容结构和版式说明 |
| `KT板展示设计-最新版.md` | KT 板最新版文案/设计稿 |
| `KT板打印稿-120x90.html` | 120x90 打印版 HTML |
| `KT板展示设计-figma-import.svg` | Figma 导入用 SVG |

这些文件当前适合作为待确认交付物；如果要同步到 GitHub，建议和生成脚本一起单独提交，避免和固件/测试改动混在一起。

## 本地预览图

`*preview*.png` 文件是本地视觉检查截图，已经在根目录 `.gitignore` 中忽略。它们适合保留在本机做对比，不作为正式仓库交付物提交。

## 维护建议

- 正式入口只保留一个当前版本，历史方案用日期或 `proposal` 命名。
- 新增导出文件时，优先在本文件补一行说明。
- 打印稿、Figma 导入稿和 Markdown 方案应保持同一主题命名，避免出现多个“最新版”但不知道哪个可用。
- 如果只是临时截图或视觉 QA 产物，文件名包含 `preview`，让 `.gitignore` 自动排除。
