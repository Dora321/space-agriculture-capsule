## 二、3分钟完整介绍话术

### 🎯 第一段：问题与灵感（45 秒）

> "人类一万年来在干一件事——**缩短育种的反馈速度**。
> 自然驯化要一万年，杂交水稻 13 年，基因编辑几个月。
> 1987 年中国第一次把水稻种子送上返回式卫星，开启了太空诱变育种——因为宇宙射线的诱变效率比地面 X 光高几十倍。到今天，中国已经培育出 240 多个航天育种品种，包括'航聚香丝苗'水稻、'鲁原 502'小麦、'太空椒'。
> 但有一个瓶颈：**种子飞一趟回来，还要在地面再种 5 到 8 代才能筛出好品种**。反馈速度被'搭载机会'卡死了。
> 我做的这个项目，是为了回答一个问题——**如果种植和验证本身就发生在太空舱里呢？** 这就是'太空农业智能种植舱'的出发点：它今天是一个全自动的地面种植舱原型，明天它的控制架构可以直接迁移到在轨育种验证平台。"

### 🔧 第二段：系统架构（60秒）

> "我们的系统分**三层架构**：
>
> **感知层**——4 类传感器：土壤湿度（电容式 ADC）、环境光照（光敏 ADC）、温湿度（DHT11）、作物选择（HS-S32-L 旋转编码器 + OLED 菜单，现场切换 8 种作物：叶菜 4 + 果菜 4）。
>
> **决策层**——**双层架构 + 三层降级**：ESP32 是舱内"飞控"，常驻**本地规则引擎**，断网断上位机也能种活作物；树莓派作为"载荷计算机"调用**云端 DeepSeek 大模型**做精细决策，把建议经 UART 发回 ESP32。决策从聪明到可靠分三层——① 树莓派调 DeepSeek → ② 树莓派阈值规则 → ③ ESP32 板上本地规则，任何一层断了下一层接住。这在'地球到火星 4 到 24 分钟延迟'的真实场景里非常关键：通信断了，舱内飞控照样自治。AI 会考虑生长阶段——苗期少浇水、果期大量水。
>
> **执行层**——双路继电器控制 12V 水泵和 12V COB 补光灯，低电平触发，带安全上限保护（水泵单次最长 60 秒、补光灯最长 120 秒、每小时最多 12 次动作、温度异常时跳过浇水）；同时采用 **Decision Plane / Action Plane 分离架构**——决策层输出多维诊断信号（缺水、缺光、高温、缺肥等），WS2812 灯条 11 颗灯珠作为「决策信号广播棒」即时广播所有信号，物理执行器仅响应 WATER/LIGHT_LOW 两种信号，其余 advisory 信号（如 TEMP_HIGH、NEED_N）通过灯条动画可视化——实现「决策能力与执行能力分离」，即使执行器未就位，诊断信息也已广播。
>
> 这套**双层**正是真实航天器的范式——ESP32 是'断了会死作物'的舱内飞控（MicroPython），树莓派是'断了只是变笨/看不到'的载荷计算机兼地面站后端，每条 AI 建议还要过 ESP32 本地安全护栏才会执行。"

### 🌱 第三段：核心亮点（60秒）

> "从'在轨育种平台'视角看，这个系统有**5 个核心能力**：
>
> **1. 多品种平行筛选能力**——`plants.json` 内置 8 种作物（叶菜 4 + 果菜 4）的独立生长阶段表、浇水阈值、施肥配方、光照需求；旋转编码器 + OLED 菜单现场切换，即选即用。
>
> **2. 全生长周期数据闭环**——系统根据种植天数自动识别苗期、营养生长期、花期、果期、采收期，不同阶段策略不同——这正是育种实验需要的"完整时间序列证据"。
>
> **3. 双层架构 + 三层降级决策**——树莓派调云端 DeepSeek 精细判断，AI 失败回退树莓派阈值规则，连树莓派都断了 ESP32 板上本地规则兜底；模拟深空通信延迟下、舱内飞控的全自治运行。
>
> **4. Decision Plane / Action Plane 分离**——决策层广播多维诊断信号（缺水、缺光、高温、缺氮/磷/钾等），物理执行器仅响应有硬件支撑的信号，其余 advisory 信号通过 WS2812 灯条动画即时广播——决策能力与执行能力分离，诊断不因执行器未就位而丢失。
>
> **5. 高密度遥测大屏**——Web 仪表盘展示 SVG 圆弧仪表、趋势曲线、决策信号面板，模拟地面飞控/育种科学家的远程数据看板。
>
> **6. 科研级可靠性**——看门狗重启、动作限频、最小间隔防抖、传感器离线降级、温度安全护栏——确保长周期育种实验不被中断。**101 个自动化测试 ALL PASS**。"

### 🎬 第四段：现场演示引导（30秒）

> "现在请看现场演示——屏幕上可以看到种植舱的实时数据：土壤湿度 35%、光照 72%、温度 24.5℃……系统刚刚自动判断土壤偏干，执行了一次 8 秒浇水，决策理由是 'soil dry'。各位也可以看到 OLED 小屏上的三页轮播：传感器页、生长状态页、系统状态页。我们随时可以切换拨码开关，让系统从养护生菜切换到养护番茄，它会立即调整所有阈值和策略——这就是'多品种平行筛选'能力的最小展示。"

## 三、评委可能的追问 + 应答

| 评委可能问 | 建议回答 |
|---|---|
| **AI决策和本地规则有什么区别？** | AI能理解更复杂的场景，比如'光照不足导致蒸发减少，所以即使土壤偏干也不急着浇水'；本地规则是硬阈值判断——土壤低于30%就浇水。我们做成三层降级：树莓派调 DeepSeek（最聪明但要网络）→ 树莓派阈值规则 → ESP32 板上本地规则（最可靠），上层断了下层接住。 |
| **为什么选ESP32而不是Arduino？** | ESP32 运算强、能跑 MicroPython、带硬件 UART 方便和树莓派通信；价格只要二三十块，性价比极高。它做舱内"飞控"，联网和 AI 交给树莓派。 |
| **为什么用MicroPython不用C++？** | MicroPython开发速度快，适合快速原型；ESP32内存足够运行Python解释器；和树莓派的 UART 协议是 JSON，Python 天然支持；我们的控制周期是60秒，Python完全够用。 |
| **断网了怎么办？** | 树莓派断网时它先用本地阈值规则给建议；连树莓派/UART 都断了，ESP32 板上本地规则全自治——土壤干燥就浇水、温度异常就跳过、传感器掉线就降级安全值，完全离线也能种活。 |
| **数据怎么传到大屏？** | ESP32 经 UART 把状态 JSON 发给树莓派，树莓派 POST 到大屏服务器，大屏网页轮询实时刷新。ESP32 自己不联网。 |
| **8种作物的数据从哪来？** | 我们查阅了农业种植手册，整理了每种作物的生长阶段、浇水阈值、施肥配方、光照需求，写成 `plants.json` 数据库。旋转编码器 + OLED 菜单现场切换 8 种（叶菜 4 + 果菜 4），即选即用。 |
| **成本多少？** | ESP32 开发板 ¥18 + 传感器约 ¥16 + 继电器 ¥6×2 + 12V 水泵 ¥18 + 12V COB 补光灯 ¥8 + WS2812 灯条 ¥8 + OLED ¥12 + 电源/降压/编码器等 ¥46，总计约 **¥140/套**（批量 10 套可压到 ¥125/套以内）。 |
| **这个系统实际种过菜吗？** | 有实际种植测试，自 `PLANTING_DATE`（详见 `config.py`）起在工位连续运行至今，累计采样次数、浇水/施肥次数请现场指向 Web 大屏右上角的 `采样次数 / 动作次数` 实时计数（这两个数由设备上报，不依赖话术口播）。<br>**答辩前 24h 务必现场操作一次**：打开大屏 → 记录当前 `read_count` 与 `action_count` → 把数字写入下面这行 → 演讲时直接说出。<br>`【现场填写】运行 ____ 天 / 采样 ____ 次 / 动作 ____ 次（_____ 月 _____ 日 _____ 时核对）` |

### 三·② 育种科学相关追问（大学教授评委高频题）

> 这 5 题大概率会被大学教授评委问到，必须提前练熟。

| 评委可能问 | 建议回答 |
|---|---|
| **你了解中国的航天育种吗？** | "了解。1987 年中国第一次用返回式卫星搭载水稻和青椒种子，是世界上少数早期开展太空育种的国家之一。2006 年的'实践八号'是专门的育种卫星，搭载了 9 类 152 种农作物种子。到今天，中国已经培育出 240 多个航天育种新品种，'航聚香丝苗'水稻和'鲁原 502'小麦是已经大规模推广的代表。航天育种是我们国家真正领先的领域之一。" |
| **宇宙射线为什么能诱变？** | "宇宙射线里有高能质子、重离子和 γ 射线，能量比地面 X 光高几个数量级。它打到 DNA 链上能直接造成断裂、点突变、片段缺失或者易位。同时微重力和高真空让细胞的 DNA 修复机制运行异常——平时能修好的损伤这时候修不全，反而保留下来变成可遗传的突变。结果就是突变频率比地面诱变高 3 到 4 倍，突变谱也更宽。" |
| **你这个项目和真正的航天育种是什么关系？** | "直接关系是：**地面原型 → 未来的在轨平台**。<br>真正的在轨育种还要解决很多额外问题——微重力下水分子聚集、根系方向紊乱、宇航员级安全标准、辐射防护。我们这套地面原型没有解决这些。<br>但我们解决的是**控制架构**：多品种平行种植、高密度数据采集、AI 实时分析、远程遥测、自治容错——这套架构在地面跑通之后，未来上轨道时不用从零开始，少走很多弯路。" |
| **现有航天育种已经成功了，你做'在轨育种'有什么必要性？** | "现有方案是'**太空诱变 + 地面验证**'，种子飞一趟回来还要在地面再种 5 到 8 代才能筛出来——反馈周期被'搭载机会'和'地面田间试种'卡住了。<br>'**在轨育种**'是'**太空诱变 + 太空验证**'——节省地面田间试种时间；而且在真实的太空环境里选育的品种，本来就是'天生适合太空'的；同时数据可以实时回传地面，让地面 AI 接力筛选，形成天地协同。<br>这条路现在还没人完整做出来——这是我十年后想做的事。" |
| **你的项目最难的部分是什么？/ 你从这个项目学到什么？** | （**这两题是软问题，是 5 年级孩子的机会题，要给"自己的话"**）<br>**最难的部分**："对我来说最难的不是写代码——代码很多是和老师一起做的。最难的是想清楚'**万一**'：万一断网怎么办、万一传感器坏了怎么办、万一温度突然飙到 40 度怎么办。平时系统看起来很简单，但要把所有'万一'都想到，比想'平时'难得多。"<br>**学到了什么**："学到了一个事——**人类一万年来都在做同一件事**，就是缩短育种的反馈速度。这件事我以前不知道。我做完这个项目之后，看新闻里讲'神舟飞船带种子上太空'就不再是看热闹，而是知道它在解决一个具体的问题。"

### 演示脚本
1. **开场**：指着 WS2812 灯条说"看这条灯——翠绿，说明生菜很舒服"，再指 OLED
2. **编码器切换**：旋转选择植物，OLED 菜单切换，大屏刷新参数，**灯条颜色梯度立刻调整**——阈值变了，灯条反映新作物的湿度区间
3. **触发浇水**：用手捏住土壤传感器（模拟干燥），**灯条从翠绿变暖红**→AI 决策→**灯条整条变黄色流水（WATER 信号动画）**→水泵动作→大屏出现 💧浇水 信号标签→**灯条恢复翠绿**
4. **信号广播演示**：指着大屏「决策信号」面板说"注意看，决策层不仅驱动水泵，还同时广播了 NEED_N（缺氮）信号——这个信号没有对应的物理执行器，但 WS2812 灯条立刻用对应动画告诉宇航员：该施肥了。这就是决策能力和执行能力分离"
5. **拔传感器**：拔掉土壤传感器，**灯条整条红闪**+OLED 弹 OFFLINE 告警——5 米外评委都能看到
6. **断网测试**：断开 WiFi，灯条颜色不变——证明本地规则照样工作
7. **大屏 2035 模式**：点击大屏右上角"切换 2035 模式"，Logo 变为 Luna Greenhouse Control Center——今天的原型，就是明天的真实
