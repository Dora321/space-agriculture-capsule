## 一、30秒电梯演讲（开场介绍）

> "各位评委好，我们的项目是**太空农业智能种植舱**——一个基于 ESP32 + 云端 AI 的全自动植物养护系统。它能像'太空植物学家'一样，**感知环境 → AI思考 → 自动浇水施肥**，让植物在无人值守的太空环境中健康生长。整个系统从传感器到执行器、从本地规则到云端AI、从OLED小屏到Web大屏，**全链路自主设计、全代码自主编写**。"

## 二、3分钟完整介绍话术

### 🎯 第一段：问题与灵感（30秒）

> "太空探索中，宇航员需要新鲜蔬菜补充营养，但太空环境下没人能天天浇水。我们的灵感来自一个问题：**能不能让AI当植物学家，7×24小时照顾太空舱里的蔬菜？** 这就是太空农业智能种植舱的出发点。"

### 🔧 第二段：系统架构（60秒）

> "我们的系统分**三层架构**：
>
> **感知层**——4类传感器：土壤湿度（电容式ADC）、环境光照（光敏ADC）、温湿度（DHT11）、作物选择（3位拨码开关，支持8种作物一键切换）。
>
> **决策层**——双引擎：优先调用**云端AI**（DeepSeek大模型），根据植物种类、生长天数、当前环境综合判断该浇水还是施肥；网络断开时自动切换**本地规则引擎**，确保离线也能正常养护。AI会考虑生长阶段——苗期少浇水、花期减水补磷钾、果期大量水钾肥。
>
> **执行层**——3个继电器控制：水泵、营养液泵、风扇，低电平触发，带安全上限保护（单次最长60秒）。
>
> 整个系统跑在 **ESP32 + MicroPython** 上，代码完全自主编写。"

### 🌱 第三段：核心亮点（60秒）

> "我们项目有**5大亮点**：
>
> **1. AI + 规则双决策引擎**：云端AI理解植物需求，本地规则保证离线可用，两者无缝切换。
>
> **2. 13种作物数据库**：每种作物有独立的生长阶段表、浇水阈值、施肥配方、光照需求。拨码开关一键切换作物类型，系统自动适配养护策略。
>
> **3. 生长阶段感知**：系统根据种植天数自动判断当前是苗期、营养生长期、花期还是果期，不同阶段给出不同的水肥策略。
>
> **4. 实时数据大屏**：Web仪表盘展示SVG圆弧仪表、趋势曲线、植物生长动画、决策流水线，支持1920×1080大屏展示。
>
> **5. 全链路安全机制**：看门狗重启、每小时动作上限、最小动作间隔防抖、传感器离线检测降级——确保系统长期稳定运行。"

### 🎬 第四段：现场演示引导（30秒）

> "现在请看现场演示——屏幕上可以看到种植舱的实时数据：土壤湿度35%、光照72%、温度24.5℃……系统刚刚自动判断土壤偏干，执行了一次8秒浇水，决策理由是'soil dry'。各位也可以看到OLED小屏上的三页轮播：传感器页、生长状态页、系统状态页。我们随时可以切换拨码开关，让系统从养护生菜切换到养护番茄，它会立即调整所有阈值和策略。"

## 三、评委可能的追问 + 应答

| 评委可能问 | 建议回答 |
|---|---|
| **AI决策和本地规则有什么区别？** | AI能理解更复杂的场景，比如'光照不足导致蒸发减少，所以即使土壤偏干也不急着浇水'；本地规则是硬阈值判断——土壤低于30%就浇水。AI更聪明但需要网络，规则更可靠但不够灵活，两者互补。 |
| **为什么选ESP32而不是Arduino？** | ESP32自带WiFi和蓝牙，能直接连云端AI；运算能力比传统Arduino强，能跑MicroPython；价格只要二三十块，性价比极高。 |
| **为什么用MicroPython不用C++？** | MicroPython开发速度快，适合快速原型；ESP32内存足够运行Python解释器；我们的AI客户端需要JSON序列化，Python天然支持；C++更适合对实时性要求极高的场景，我们的控制周期是60秒，Python完全够用。 |
| **断网了怎么办？** | 自动切换本地规则引擎，土壤干燥就浇水，营养液到期就施肥，完全离线也能正常运行。 |
| **数据怎么传到大屏？** | ESP32通过WiFi把状态JSON POST到云端服务器，大屏网页轮询获取数据，实时刷新。 |
| **13种作物的数据从哪来？** | 我们查阅了农业种植手册，整理了每种作物的生长阶段、浇水阈值、施肥配方、光照需求，写成JSON数据库。 |
| **成本多少？** | ESP32开发板30元 + 传感器约20元 + 继电器模块15元 + 水泵营养液泵约30元 + OLED屏幕10元 + 其他配件约20元，总计约125元。 |
| **这个系统实际种过菜吗？** | 有实际种植测试，系统从5月1日开始运行，目前已运行超过18天，累计自动浇水X次、施肥X次（用实际数据填入）。 |

## 六、现场演示 Checklist

### 硬件准备
- [ ] ESP32开发板 + USB供电
- [ ] 传感器接线正常（土壤/光照/DHT/拨码）
- [ ] 继电器 + 水泵/营养液泵接线正常
- [ ] OLED显示正常
- [ ] WiFi连接正常
- [ ] 状态LED正常（绿=正常/黄=执行中/红=故障）

### 软件准备
- [ ] MicroPython固件已烧录
- [ ] config.py中WiFi密码正确
- [ ] AI API Key有效
- [ ] Web大屏浏览器已全屏打开
- [ ] Demo模式已关闭（展示真实数据更好）

### 演示脚本
1. **开场**：指着OLED说"这是种植舱的实时状态"
2. **拨码切换**：拨动开关，OLED和大屏同步切换作物类型
3. **触发浇水**：用手捏住土壤传感器（模拟干燥），观察AI决策→水泵动作→OLED和大屏更新
4. **断网测试**：断开WiFi，展示本地规则接管
5. **大屏展示**：引导评委看大屏上的趋势曲线和植物生长动画
